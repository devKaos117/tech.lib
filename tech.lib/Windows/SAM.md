Windows stores hashed user passwords in the [_Security Account Manager_](https://en.wikipedia.org/wiki/Security_Account_Manager) (SAM) database file, which is used to authenticate local or remote users

> [!info] To deter offline SAM database password attacks, Microsoft introduced the _SYSKEY_ feature in Windows NT 4.0 SP3, which partially encrypts the SAM file. The passwords can be stored in two different hash formats: [_LAN Manager_](https://en.wikipedia.org/wiki/LAN_Manager) (LM) and NTLM. LM is based on [_Data_Encryption_Standard_](https://en.wikipedia.org/wiki/Data_Encryption_Standard) (DES), and is known to be very weak. For example, passwords are case insensitive and cannot exceed fourteen characters. If a password exceeds seven characters, it is split into two strings, each hashed separately. LM is disabled by default beginning with Windows Vista and Windows Server 2008
